1. Explique 3 patrones utilizados en programación y proporcione un ejemplo de uso para cada uno.

Patrón Singleton: Garantiza una única instancia de una clase.

public class Singleton {
    private static Singleton instancia;
    private Singleton() {}
    public static Singleton getInstancia() {
        if (instancia == null) instancia = new Singleton();
        return instancia;
    }
}

Patrón Observador: Permite que múltiples objetos sean notificados de cambios en otro objeto.

interface Observador {
    void actualizar(String mensaje);
}

class Sujeto {
    private List observadores = new ArrayList<>();
    void agregarObservador(Observador o) { observadores.add(o); }
    void notificar(String mensaje) {
        for (Observador o : observadores) o.actualizar(mensaje);
    }
}

Patrón Fábrica: Facilita la creación de objetos sin especificar la clase exacta.

class Fabrica {
    public static Producto crearProducto(String tipo) {
        if (tipo.equals("A")) return new ProductoA();
        else return new ProductoB();
    }
}

2. ¿Cuál es la diferencia entre una base de datos relacional y una no relacional? Proporcione 2 ejemplos.

Relacionales: Basadas en SQL, organizadas en tablas. Ejemplo: MySQL, PostgreSQL.

-- Ejemplo en MySQL
CREATE TABLE Empleados (
    id INT PRIMARY KEY,
    nombre VARCHAR(100),
    salario DECIMAL(10,2)
);

INSERT INTO Empleados (id, nombre, salario) VALUES (1, 'Juan', 2500.00);
SELECT * FROM Empleados WHERE salario > 2000;

No Relacionales: Basadas en documentos JSON o clave-valor. Ejemplo: MongoDB, Firebase.

// Ejemplo en MongoDB
db.empleados.insertOne({ "id": 1, "nombre": "Juan", "salario": 2500 })
db.empleados.find({ "salario": { "$gt": 2000 } })

3. En programación web, al enviar los datos de formularios, ¿es mejor usar el método GET o POST? Explique y dé 2 ejemplos.

¿Cuál es mejor? En la mayoría de los casos, POST es la mejor opción, ya que es más seguro y adecuado para enviar datos sensibles como contraseñas o información personal. Sin embargo, GET es útil cuando solo se necesitan recuperar datos sin realizar cambios en el servidor.

GET: Se utiliza cuando se quiere enviar datos a través de la URL. Es útil para búsquedas y accesos rápidos a recursos. Sin embargo, tiene limitaciones de tamaño y no es seguro para datos sensibles.

<a href="buscar.php?query=producto">Buscar Producto</a>

Ejemplo en PHP con GET:

<?php
if(isset($_GET['query'])) {
    echo "Buscando: " . htmlspecialchars($_GET['query']);
}
?>

POST: Se usa cuando se necesita enviar información de manera segura, sin mostrar los datos en la URL. Es ideal para formularios de autenticación o envío de información sensible.

<form action="login.php" method="POST">
    <input type="text" name="usuario">
    <input type="password" name="clave">
    <input type="submit" value="Ingresar">
</form>

Ejemplo en PHP con POST:

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $usuario = htmlspecialchars($_POST['usuario']);
    echo "Bienvenido, " . $usuario;
}
?>

5. En SQL Server, ¿cuál es la diferencia entre las consultas SELECT al utilizar LEFT JOIN, RIGHT JOIN e INNER JOIN? Explique y proporcione un ejemplo.

LEFT JOIN: Devuelve todos los registros de la tabla izquierda y los coincidentes de la derecha. Si no hay coincidencias, devuelve NULL.

RIGHT JOIN: Devuelve todos los registros de la tabla derecha y los coincidentes de la izquierda. Si no hay coincidencias, devuelve NULL.

INNER JOIN: Solo devuelve los registros que tienen coincidencias en ambas tablas. 

-- Ejemplo en SQL Server
SELECT empleados.nombre, departamentos.nombre
FROM empleados
LEFT JOIN departamentos ON empleados.depto_id = departamentos.id;
-- Ejemplo con RIGHT JOIN
SELECT empleados.nombre, departamentos.nombre
FROM empleados
RIGHT JOIN departamentos ON empleados.depto_id = departamentos.id;
-- Ejemplo con INNER JOIN
SELECT empleados.nombre, departamentos.nombre
FROM empleados
INNER JOIN departamentos ON empleados.depto_id = departamentos.id;

7. Describa una solución para resolver un sistema de inventario en un almacén.

¿Cómo manejaría las actualizaciones y consultas concurrentes?

Para manejar actualizaciones y consultas concurrentes en una base de datos de inventario, se pueden utilizar bloqueos optimistas y transacciones. Esto previene inconsistencias al permitir que múltiples usuarios accedan a los datos sin bloquear innecesariamente los registros.

-- Ejemplo en SQL para transacciones
START TRANSACTION;
UPDATE productos SET cantidad = cantidad - 1 WHERE id = 10;
COMMIT;

¿Cómo diseñaría un sistema para detectar y notificar automáticamente cuando un producto esté por debajo del nivel mínimo de inventario?

Se puede implementar un proceso automatizado que verifique periódicamente los niveles de stock y envíe notificaciones cuando un producto esté por debajo del umbral mínimo.

-- Ejemplo en SQL para detectar inventario bajo
SELECT * FROM productos WHERE cantidad < nivel_minimo;
// Ejemplo en Python para enviar alerta automática
if producto["cantidad"] < producto["nivel_minimo"]:
